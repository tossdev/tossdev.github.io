구매자 회원 가입 조건

만 14세 이상이면 누구나 회원 가입 후 토스 결제를 이용할 수 있습니다.
토스머니는 자신의 은행 계좌를 통해 자동충전할 수 있고, 토스카드는 본인 소유의 카드를 등록해서 사용할 수 있습니다.

토스 머니
  • 지원 은행 : 우리 / KB국민 / IBK기업 / 부산 / 경남 / 우체국 / 광주 / 전북 / 새마을 / NH농협 / 신협 / SC / 대구 / KDB산업 / 제주 / KEB하나 (구 외환은행 포함) / 수협 / 신한 / 카카오뱅크
  • 지원 증권사 : 키움 / 삼성 / NH투자 / 대신 / 신한금융투자 / 메리츠종금 / 유진투자
토스 카드
  • 지원 카드사 : 롯데 / 비씨 / 삼성 / 신한 / 하나 (그 외 카드사는 현재 준비 중입니다)

구매자 결제 한도

일/월 한도

성인 회원
  • 회원당 1회 2,000,000원 까지 결제 가능 (이를 초과하는 결제는 생성할 수 없습니다)
  • 회원당 1일 누적 2,000,000원 까지 결제 가능
미성년자 회원 (만 19세 미만)
  • 회원당 1회 300,000원 까지 결제 가능
  • 회원당 1일 누적 300,000원 까지 결제 가능
  • 회원당 1개월 누적 1,000,000원 까지 결제 가능

* 단, 위 한도는 표준 한도이며, 각 가맹점별/고객별 판매 품목이나 Risk 등급에 따라 결제 가능 한도를 제한할 수 있습니다.

취소/환불 시 한도 회복

당일 취소/환불 시 일 한도 회복되며, 당월 취소/환불 시 월 한도 회복

예시
  • 6/29 결제 500,000원 → 6/29 당일 환불 완료 시
    • 6/29 일 결제 한도 +500,000
    • 6월 월 결제 한도 +500,000
  • 6/29 결제 500,000원 → 6/30 환불 완료 시
    • 6/30 일 결제 한도 변함 없음
    • 6월 월 결제 한도 +500,000
  • 6/30 결제 500,000원 → 7/1 환불 완료 시
    • 7/1 일 결제 한도 변함 없음
    • 7월 월 결제 한도 변함 없음

결제 취소

정의

결제는 생성되었지만 아직 결제가 완료되지 않은 상태에서 더 이상 결제를 진행할 수 없는 상태로 만드는 행위

취소 발생 상황

  • 구매자가 토스 앱을 통해 결제 취소하는 경우
  • 판매자가 API 또는 관리자를 통해 취소하는 경우
  • 결제 생성 후 2분 내에 결제할 사람의 전화번호가 입력되지 않은 경우
  • 결제 생성 시 지정한 만료 기간 내에 결제가 완료되지 않은 경우 (최대 1시간)

환불

정의

결제를 생성하고 완료한 이후에 지불한 금액을 다시 돌려받는 행위

환불 과정

환불 요청 시, 환불 금액이 해당 상점의 환불 한도보다 작으면 환불 완료 처리하고, 크면 거부 처리합니다.

상점의 환불 한도

각 '상점'별로 '환불 한도'가 부여되며, 환불 한도 내의 환불 신청은 즉시 처리됩니다. ('환불 한도'만 확보하면 정산 여부나 결제 시기와 상관 없이 유연한 환불이 가능합니다)

(상세 정책)

  • 환불 한도는 각 '상점'별로 부여한다.
  • 환불 한도는 매일 자정(00:00)에 기본값으로 초기화된다.
  • 결제 완료 시, 결제 금액 만큼 환불 한도가 늘어난다.
  • 환불 완료 시, 환불 금액 만큼 환불 한도가 줄어든다.
  • 환불 한도 기본값은 상점의 거래량에 따라 늘어나거나 줄어든다. (급히 높여야 할 경우, 토스 고객센터로 연락바랍니다. 1599-4905)

전액/부분 환불 가능 여부

  • 일반 결제의 경우, 전액 환불과 부분 환불 모두 가능
  • 에스크로 결제의 경우, 정산 이후에만 부분 환불 가능
    • 에스크로 진행중인 상태 (요청/해제/거부) → 전액 환불만 가능
    • 에스크로 상태가 끝나고 정산이 완료 → 부분 환불 가능
  • 토스를 탈퇴할 경우 환불 불가

환불 시 수수료 처리

  • 전액 환불하는 경우 : 가맹점으로부터 받았던 결제 대행 수수료(+VAT)를 모두 돌려주고, 환불 처리 수수료(+VAT)를 부과
  • 부분 환불하는 경우
    • 해당 부분 환불 후 남은 거래금액이 있을 경우 : 환불 수수료 부과 X, 결제 대행 수수료 및 부과세 반환 X
    • 해당 부분 환불 후 남은 거래금액이 없는 경우 : 환불 수수료 부과 O, 결제 대행 수수료 및 부과세 전액 반환
  • 환불 처리 수수료는 아래 '수수료' 절 참고

환불 금액은 어디로 가는가?

환불 금액은 토스머니로 돌아갑니다. 따라서 따로 '환불 계좌'를 수집하지 않아도 됩니다.

환불 가능 기간

  • 일반 결제는 결제 승인일로부터 1년까지
  • 에스크로 결제는 구매확정일로부터 180일까지

결제 대행 수수료

표준 수수료 정책

  • 수수료 : 결제 수단 상관없이 결제 금액의 3%
  • 환불 처리 수수료 : 400원 (전액 환불 시에만 부과)
  • (부가세는 별도이며 수수료는 업종/판매물품 별로 다를 수 있습니다)

수수료의 부가세

결제 대행 수수료의 10%를 부가세로 책정한다. (매출건별)

수수료 적용 시점

  • 일반 거래 : 승인일자

원단위 미만의 수수료 및 부가세 처리

  • 결제 대행 수수료는 소수점 부분을 '버림' (ex. 1100.7원 → 1100원)
  • 부가세는 소수점 부분에서 '반올림' (ex. 110.7원 → 111원)

에스크로 결제

에스크로 결제가 필요하신 경우 별도 문의해주세요.

에스크로 결제는 구매자 보호를 위한 결제안전장치입니다. 결제가 끝났으나 물품을 받지 못하거나 올바르지 않은 물품을 받았을 때 결제 대금을 잃어버리는 피해를 막기 위해 제공하는 서비스로, 결제 대금을 제3자인 '토스'가 예치하고 있다가 상품배송이 끝나고 구매자가 구매를 확정하면 지급하는 결제 방식입니다.

구매자는 안심하고 구매할 수 있고, 판매자는 신뢰를 얻을 수 있는 결제 수단으로, '온라인 상'에서 '유형의 물품'을 판매하는 경우 구매자에게 에스크로 결제 선택권을 반드시 제공해야 합니다.

전체 흐름

  • 에스크로 결제의 경우, 결제 성공 후 바로 '배송등록 대기 중'상태로 변경되어 결제금액 보호가 시작됩니다.
  • 가맹점이 물품을 발송하고 '배송등록'합니다. 구매자는 물품을 받은 후 '구매확정'함으로써 결제가 완료됩니다.
  • 구매자는 거래에 이상이 있는 경우, '지급거부'함으로써 결제대금을 보호받을 수 있습니다.

'배송등록 대기 중' 상태

  • 이 상태에서 가맹점 측에서는 '배송 등록' 또는 '전액 환불' 처리할 수 있습니다. (부분환불 불가)
  • 가맹점측에서 물품을 배송한 후, '배송 등록'을 실시하면 '구매확정 대기 중'상태로 변경됩니다. (구매자에게는 배송 시작 안내 push 또는 SMS 발송)
  • 이 단계에서 가맹점이 120일간 아무런 행동을 하지 않으면 '구매자 지급 거부'상태로 변경됩니다.

'구매확정 대기 중' 상태

  • 이 상태에서 구매자는 '구매확정' 또는 '지급거부'할 수 있으며, 가맹점은 '전액 환불' 처리할 수 있습니다. (부분 환불 불가)
  • 구매자가 '구매확정'을 하면 '에스크로 승인' 상태로 변경되고, 곧 가맹점 대금 정산이 시작됩니다.
  • 구매자가 '지급거부'하면 '구매자 지급 거부'상태로 변경됩니다. (이 상태가 해결될 때 까지 대금 정산은 이루어지지 않습니다)
  • 구매자가 7영업일간 아무런 행동을 하지 않으면 자동으로 구매확정 처리합니다. (자동 처리 전날 경고 push 또는 SMS 발송)

'구매자 지급 거부' 상태

  • 이 상태에서 가맹점은 '구매확정 재요청'하거나 '전액 환불' 처리할 수 있습니다. (부분 환불 불가)
  • 가맹점이 구매자와 문제를 해결한 후, '구매확정 재요청'할 수 있습니다. 재요청 시 해당 결제 건은 '구매확정 대기 중'상태로 변경되며, 구매확정 요청 push/SMS가 재발송됩니다.
  • 구매자와의 문제가 해결되지 않은 채로 '구매확정 재요청'할 경우, 구매자에 의해 다시 '지급거부'처리되므로 반드시 구매자와 연락 후 재요청하시기 바랍니다.

정산 / 지급

정산 및 지급 과정

상점 측에 총 얼마를, 언제 지급할지 계산하고 계산된 금액을 가맹점에 지급하는 과정입니다.

  1. 정산 집계 (일마감)
    • 매일 지정된 시간에 지급할 금액과 날짜를 정합니다. (상점별)
    • 집계 시간 : 새벽 2~3시 사이
    • 집계 대상 : 지난 집계 이후 결제/환불 완료 처리된 건 + 지난 집계에서 정산되지 않은 이연 건
    • 지급할 금액 : 집계 대상이 되는 모든 건의 금액을 더하여 구합니다. (단, 지급할 금액이 '-'가 되면, 일부 환불 건들을 '이연'처리함)
  2. 금액 대사 및 보정/확정
    • 매 영업일마다 비바 리퍼블리카 정산 담당자가 각 상점에 지급할 금액을 확인하고, 필요 시 보정을 진행한 후 최종 승인합니다. (영업일 15시 전까지)
  3. 정산금액 가맹점 지급
    • 승인한 정산 금액을 각 상점의 정산 계좌로 입금합니다.

집계 및 정산 예시

토스에서 정산 집계가 어떻게 이루어지는지, 예시와 함께 자세히 설명드리겠습니다.
2019. 4. 19 새벽, 정산 집계하는 상황을 가정하고 살펴봐주세요.

[상점 A의 Case : 일반적인 결제/환불 처리]
  • 상점 A에선 4월 15일에 아래와 같은 일이 있었습니다.
    • 68,100원 토스 카드로 결제
    • 220,000원 토스 머니로 결제했다가 곧 전액 환불 처리
    • 98,000원 토스 카드로 결제했다가 그 중 12,000원만 환불 처리
  • 집계 결과는 아래와 같습니다.
    번호 승인확정일 종류 결제금액 수수료 부가세 지급액 정산 상태 지급예정일
    1 4/15 결제 68,100 -2,043 -204 65,853 - -
    2 4/15 결제 220,000 -6,600 -660 212,740 - -
    3 4/15 환불 -220,000 6,600 660 -212,740 - -
    4 4/15 환불 수수료 -400 -40 -440 - -
    5 4/15 결제 98,000 -2,940 -294 94,766 - -
    6 4/15 환불 (부분) -12,000 -12,000 - -
    • 1번 - 68,100원의 결제금액에서 수수료와 수수료에 대한 부가세를 제외한 65,853원을 지급합니다. 수수료는 68,100원의 3%에 해당하는 2,043원이고, 수수료에 대한 부가세는 2,043원의 10%인 204.3원입니다만, '원단위 미만의 수수료 및 부가세 처리' 정책에 따라 소수점 부분에서 반올림하여 204원으로 처리하였습니다.
    • 2, 5번 - '결제'건으로 1과 동일하게 처리되었습니다.
    • 3번 - 2번 결제에 대해 전액 환불이 발생하여 결제 금액을 모두 차감하고, 받았던 수수료와 부가세 모두 돌려드립니다.
    • 4번 - 환불 수수료는 없습니다.
    • 6번 - 부분 환불이 발생하여 환불한 금액 만큼 결제 금액을 차감합니다. 이 경우에도 환불 수수료는 없습니다.
  • 정산 결과는 아래와 같습니다.
    번호 승인확정일 종류 결제금액 수수료 부가세 지급액 정산 상태 지급예정일
